# AiMall 电商系统需求文档（V1）

## 1. 项目概述

### 1.1 项目目标
建设一款 Web 电商系统，包含：
- PC 端商城（用户端）
- PC 端管理后台（管理端）

本期目标是先完成核心商品、分类、客户、行为轨迹、推荐展示与购物车能力，支持管理端维护数据、商城端浏览与下单前流程（结算页可先保留占位）。

### 1.2 技术约束（已确定）
- 后端：Python + Django
- 前端：Vue + TypeScript + HTML
- 数据库：MySQL

### 1.3 复用要求（强约束）
必须复用同级项目 `GhostFit` 的前端资源与交互模式（尤其是增删改查列表、抽屉表单、详情抽屉、行为轨迹展示等），避免重复设计需求与重复造轮子。

复用重点（来自 `GhostFit/Dev-Frontend`）：
- 通用列表骨架：`ListPageScaffold.vue`
- 表格渲染：`SchemaTable.vue`
- 搜索表单：`SchemaSearchForm.vue`
- 抽屉表单：`SchemaDrawerForm.vue`
- 详情抽屉布局：`DetailDrawerLayout.vue`
- 行为轨迹时间线：`BehaviorTimeline.vue`
- 布局与左侧菜单：`src/components/layout/*`
- 路由与菜单配置模式：`src/router/routes.ts`
- API 封装与状态管理结构：`src/api/*`、`src/stores/*`

## 2. 建设范围与阶段说明

### 2.1 本期范围（V1）
- 管理端登录页（固定开发账号密码）
- 管理端三层左侧菜单
- 商品管理（商品列表、类别列表、智能推荐、线上预览）
- 客户管理（客户列表、客户详情抽屉、行为轨迹）
- 商城首页（分类、搜索、推荐、商品列表、购物车入口）
- 商品列表页（按搜索/分类查看）
- 购物车页（勾选、价格汇总、结算按钮）
- 商品详情页：先完成页面结构与静态布局，功能细化后续补充

### 2.2 暂不在本期实现（建议）
- 支付、订单履约、退款、物流
- 真实推荐模型训练平台（先做规则/简单算法）
- 多角色权限系统（先保留管理员单角色）
- 多商户/多租户

## 3. 角色定义
- 管理员：维护商品、分类、客户数据，查看推荐结果与商城线上预览。
- 商城用户：搜索商品、浏览分类、查看推荐、加入购物车、查看购物车。

## 4. 管理端（PC 管理后台）需求

### 4.1 UI 风格与基础能力
- 整体风格：蓝白主题
- 登录校验：账号密码登录
- 开发期间固定账号密码：
  - 账号：`admin`
  - 密码：`123456`
- 左侧菜单支持 3 级菜单展示与展开收起
- 页面通用能力（需复用 GhostFit 模式）：
  - 列表页搜索区
  - 表格列表
  - 分页
  - 抽屉新增/编辑
  - 删除确认
  - 详情抽屉

### 4.2 菜单结构
- 商城
  - 商品管理
    - 商品列表
    - 类别列表
    - 智能推荐
    - 线上预览
- 客户
  - 客户管理
    - 客户列表

说明：该结构为 3 级菜单，需在路由与菜单元信息中直接支持。

### 4.3 商城 / 商品管理 / 商品列表

#### 4.3.1 功能目标
实现商品信息增删改查，支持规格（SKU）与图片上传。

#### 4.3.2 商品字段（V1 建议）
- 基础信息：
  - 商品ID（系统生成）
  - 商品名称（必填）
  - 商品副标题（可选）
  - 商品分类（关联类别，必填）
  - 商品主图（必填）
  - 商品图集（可选，多图）
  - 商品简介（可选）
  - 商品详情（后续商品详情页会使用，可先文本/富文本占位）
- 销售信息：
  - 默认规格（展示用）
  - 默认价格（分/元统一后端存储）
  - 销量
  - 浏览量
  - 上下架状态
  - 排序值
- 规格信息（SKU）：
  - 规格名称（如：颜色/容量）
  - 规格项（如：黑色、128G）
  - SKU 编码（可选）
  - SKU 价格
  - SKU 库存
  - SKU 图片（可选）

#### 4.3.3 列表页能力
- 按商品名称搜索
- 按分类筛选
- 按状态筛选（上架/下架）
- 分页查询
- 新增商品（跳转页或抽屉均可，建议复用 GhostFit 商品编辑页结构）
- 编辑商品
- 删除商品
- 上下架切换
- 批量操作（建议复用 GhostFit 批量操作模式）

### 4.4 商城 / 商品管理 / 类别列表

#### 4.4.1 功能目标
实现商品类别增删改查，类别支持 3 级树形结构。

#### 4.4.2 字段建议
- 分类ID
- 分类名称（必填）
- 父分类ID（空表示一级）
- 分类层级（系统计算，1~3）
- 图标（可选）
- 排序
- 是否启用
- 商品数量（统计）

#### 4.4.3 业务规则
- 最多支持 3 级分类
- 3 级分类不可再新增子分类
- 删除分类前校验是否存在子分类/关联商品（可限制删除或提示迁移）
- 管理端分类树数据需与商城端首页分类导航实时同步（至少接口级同步）

### 4.5 商城 / 商品管理 / 智能推荐

#### 4.5.1 功能目标
展示“针对每个用户的推荐商品信息”，作为推荐结果管理与验证页面。

#### 4.5.2 页面内容（V1）
- 用户筛选（姓名/手机号）
- 推荐结果列表（按用户维度）
- 展示字段建议：
  - 用户信息（姓名、手机号）
  - 推荐商品（TopN，例如 Top10）
  - 推荐原因（如：近期搜索、浏览、加购、同类偏好）
  - 推荐分值（可选）
  - 推荐时间
- 支持“刷新推荐”按钮（触发实时规则计算或重新拉取）

#### 4.5.3 推荐依据
基于用户行为轨迹数据：
- 搜索关键词
- 点击类别
- 点击商品详情
- 加入购物车

### 4.6 商城 / 商品管理 / 线上预览
- 支持一键跳转至 PC 商城用户端首页（新窗口打开）
- 建议支持环境配置（开发/测试/生产商城域名）

### 4.7 客户 / 客户管理 / 客户列表

#### 4.7.1 功能目标
实现客户信息增删改查；点击客户名弹出抽屉，展示客户详情与行为轨迹。

#### 4.7.2 客户信息字段（用户指定）
- 姓名
- 手机号
- 年龄
- 爱好
- 地址

补充建议字段（系统字段）
- 客户ID
- 创建时间
- 更新时间
- 最近活跃时间

#### 4.7.3 列表页能力
- 搜索（姓名/手机号）
- 分页
- 新增客户
- 编辑客户
- 删除客户
- 点击客户名打开详情抽屉

#### 4.7.4 客户详情抽屉内容
- 基本信息（姓名、手机号、年龄、爱好、地址等）
- 行为轨迹（时间线）

#### 4.7.5 行为轨迹定义（用于推荐）
记录用户在 Web 商城的行为，示例：
- 搜索了 `xxx` 商品
- 点击了 `xxx` 类别
- 点击了 `xxx` 商品详情
- 将 `xxx` 商品加入了购物车

行为轨迹将作为智能推荐的依据，需至少记录：
- 用户ID
- 行为类型
- 行为对象类型（商品/分类/关键词）
- 行为对象ID（如有）
- 行为对象名称/关键词快照
- 时间戳
- 扩展信息（JSON，可选）

## 5. 商城端（PC 用户端）需求

### 5.1 首页

#### 5.1.1 顶部区域
- 顶部中央搜索框
  - 支持按商品名称搜索商品信息
  - 回车或点击搜索后跳转商品列表页
- 右上角图标
  - 购物车 icon：点击跳转购物车页面
  - 用户头像 icon：点击后可展示用户菜单（V1 可先做占位）

#### 5.1.2 左侧分类导航（重点）
- 展示商品类别列表（与管理端类别列表同步）
- 支持 3 级分类展示
- 鼠标悬停在 1 级分类名称时，自动平滑弹出抽屉/浮层，展示该分类下 2 级、3 级分类
- 点击任意分类跳转商品列表页并带分类筛选条件

#### 5.1.3 为你推荐区域
- 根据推荐算法推荐偏好商品 4 个
- 商品展示内容：图片、名称、默认规格、价格等核心字段
- 右上角有“换一换”按钮
- 点击“换一换”后推荐另外 4 个商品
- 允许连续多次点击“换一换”（每次都返回不同/尽量不重复结果）

#### 5.1.4 商品列表区域（首页随机流）
- 随机展示不同类别商品 20 个
- 一排展示 4 个
- 页面下拉时继续加载更多商品（无限滚动/加载更多均可）
- 商品卡片固定展示：
  - 图片
  - 名称
  - 默认规格
  - 价格
  - 销量
  - 浏览量
- 点击商品名称/价格/图片跳转商品详情页

### 5.2 商品列表页
- 来源场景：
  - 首页搜索进入
  - 分类点击进入
- 功能要求：
  - 显示商品列表
  - 支持按商品名称搜索（可复用顶部搜索）
  - 支持分类条件展示/切换
  - 分页或无限滚动（二选一，建议分页更稳）
  - 点击商品进入详情页

### 5.3 购物车页面
- 显示当前用户加入购物车的商品列表
- 每个商品展示：
  - 图片
  - 名称
  - 所选规格
  - 单价
  - 数量
  - 总价（单行）
- 支持勾选多个商品
- 底部自动计算已勾选商品总价
- 右下角“结算”按钮（V1 可跳转占位页或提示“暂未开放”）

### 5.4 商品详情页
- 当前阶段要求：先把页面做出来（结构与样式优先）
- 功能细节（加入购物车、规格切换、详情图文等）在下一阶段补充

建议本期页面至少包含：
- 商品主图/图集区域
- 商品名称、价格、销量、浏览量
- 默认规格展示（或规格选择占位）
- 商品简介
- 加入购物车按钮（可占位）
- 商品详情内容区（占位）

## 6. 非功能需求

### 6.1 可用性
- 管理端页面操作反馈明确（保存成功/失败、删除确认）
- 抽屉关闭前未保存提示（复用 GhostFit 交互）
- 上传图片失败需提示原因

### 6.2 性能
- 分类树、推荐、首页商品流接口应支持分页/限流
- 首页随机商品流接口支持按批次返回（20 条/次）
- 图片使用压缩与缩略图

### 6.3 安全（V1 最低要求）
- 管理端登录鉴权
- 接口鉴权与登录态校验
- 密码不得明文存储（即使开发账号也建议加密保存）
- 输入校验与基础防注入

### 6.4 日志与可追踪性
- 用户行为轨迹必须落库
- 管理端关键操作建议记录审计日志（商品/分类/客户的增删改）

## 7. 验收标准（V1）

### 7.1 管理端验收
- 能使用 `admin / 123456` 登录后台（开发环境）
- 左侧菜单能展示 3 级结构
- 商品列表完成增删改查，支持规格与图片上传
- 分类列表完成 3 级分类增删改查
- 客户列表完成增删改查
- 点击客户名可打开详情抽屉并显示行为轨迹
- 智能推荐页面可按用户展示推荐结果
- 线上预览按钮能跳转商城首页

### 7.2 商城端验收
- 首页顶部搜索可用
- 首页右上角购物车入口可跳转
- 左侧分类导航支持 3 级悬停展开
- 为你推荐区域每次展示 4 个，可持续“换一换”
- 首页商品列表区域随机展示 20 个并支持继续加载
- 商品卡片字段显示完整
- 购物车页面可勾选、汇总总价、点击结算
- 商品详情页页面结构完成（本期先不要求完整业务）
