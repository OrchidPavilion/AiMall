# GhostFit 管理后台 - 前端技术设计文档

## 1. 技术架构选型

### 1.1 核心框架
- **Vue 3**: Composition API + `<script setup>` 语法
- **TypeScript**: 强类型支持，提升代码健壮性
- **Vite**: 快速构建工具，支持热更新，优化开发体验

### 1.2 UI 组件库
**Element Plus** (v2.3+)
- 理由：成熟稳定、中文文档完善、主题定制灵活、符合蓝白主题需求
- 组件覆盖：表格、表单、弹窗、导航、通知等全套 B 端组件
- 主题定制：通过 SCSS 变量覆盖实现蓝白配色

### 1.3 状态管理
**Pinia** (v2.1+)
- 理由：Vue 3 官方推荐、TypeScript 友好、轻量简洁
- 模块划分：
  - `auth`: 认证状态（用户信息、Token、权限）
  - `app`: 应用级状态（主题、侧边栏折叠、面包屑）
  - `customer`: 客户管理状态（筛选条件、选中项）
  - `order`: 订单管理状态（筛选条件、批量操作）
  - `product`: 商品管理状态

### 1.4 路由管理
**Vue Router** (v4.2+)
- 路由模式：History Mode
- 路由守卫：权限校验、登录拦截
- 路由配置：按模块分组，支持懒加载

### 1.5 HTTP 客户端
**Axios** (v1.6+)
- 封装拦截器：统一添加 Token、处理错误、请求取消
- 请求超时：15s
- 响应拦截：统一错误码处理、业务异常抛出
- 请求取消：路由切换时取消未完成请求

### 1.6 样式方案
- **SCSS**: 嵌套语法、变量、混合
- **CSS Modules**: 组件级样式隔离
- **主题变量** (variables.scss):
  ```scss
  // 主色调

  $primary-light: #40a9ff;

  ```

### 1.7 工具库
- **date-fns**: 日期格式化 (v3.3+)
- **lodash-es**: 工具函数 (v4.17+)
- **vue-i18n**: 国际化 (可选，v9.8+)
- **nprogress**: 页面加载进度条 (v0.2+)

### 1.8 开发工具

### 3.2 路由守卫

```typescript
// router/guards.ts
router.beforeEach(async (to, from, next) => {
  // loading 开始
  NProgress.start();
  
  // 确定是否需要认证
  const requiresAuth = to.matched.some(record => !record.meta.requiresGuest);
  
  // 已登录用户访问登录页，重定向到首页
  if (!requiresAuth && store.auth.isLoggedIn) {
    next('/dashboard');
    return;
  }
  
  // 检查认证状态
  if (requiresAuth) {
    if (!store.auth.isLoggedIn) {
      next('/login');
      return;
    }
    
    // 权限检查
    const hasPermission = await store.permission.hasPermission(to.meta.permission);
    if (!hasPermission) {
      next('/403'); // 或无权限页面
      return;
    }
  }
  
  next();
  NProgress.done();
});
```

## 4. 状态管理设计 (Pinia)

### 4.1 auth.store.ts - 认证管理

```typescript
export const useAuthStore = defineStore('auth', {
  state: () => ({
    token: localStorage.getItem('token') || '',
    userInfo: null as UserInfo | null,
    roles: [] as string[],
    permissions: [] as string[]
  }),
  
  getters: {
    isLoggedIn: (state) => !!state.token,
    isSuperAdmin: (state) => state.roles.includes('SUPER_ADMIN')
  },
  
  actions: {
    async login(credentials: LoginCredentials) {
      const { data } = await api.login(credentials);
      this.token = data.token;
      this.userInfo = data.user;
      localStorage.setItem('token', data.token);
    },
    
    async fetchUserInfo() {
      const { data } = await api.getUserInfo();
      this.userInfo = data;
      this.roles = data.roles;
      this.permissions = data.permissions;
    },
    
    logout() {
      this.$reset();
      localStorage.removeItem('token');
      router.push('/login');
    },
    
    updateToken(token: string) {
      this.token = token;
      localStorage.setItem('token', token);
    }
  }
});
```

### 4.3 permission.store.ts - 权限管理

```typescript
export const usePermissionStore = defineStore('permission', {
  state: () => ({
    menus: [] as MenuItem[],
    permissionsMap: new Map<string, boolean>()
  }),
  
  getters: {
    hasPermission: (state) => {
      return (permission: string) => {
        if (state.permissionsMap.has(permission)) {
          return state.permissionsMap.get(permission);
        }
        // 超级管理员拥有所有权限
        return state.permissionsMap.has('*') || permission === '';
      };
    }
  },
  
  actions: {
    async setMenus(menus: MenuItem[]) {
      this.menus = menus;
    },
    
    setPermissions(permissions: string[]) {
      this.permissionsMap.clear();
      permissions.forEach(p => {
        this.permissionsMap.set(p, true);
      });
    }
  }
});
```

## 5. API 封装与对接

### 5.1 Axios 封装

```typescript
// api/request.ts
import axios from 'axios';
import { message } from '@/utils/message';
import { useAuthStore } from '@/stores/auth.store';

const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || '/api/v1',
  timeout: 15000,
  headers: {
    'Content-Type': 'application/json'
  }
});

// 请求拦截器
request.interceptors.request.use(
  (config) => {
    const authStore = useAuthStore();
    if (authStore.token) {
      config.headers.Authorization = `Bearer ${authStore.token}`;
    }
    
    // 可选：添加请求ID用于追踪
    config.headers['X-Request-ID'] = crypto.randomUUID();
    
    return config;
  },
  (error) => Promise.reject(error)
);

// 响应拦截器
request.interceptors.response.use(
  (response) => {
    const { data } = response;
    
    // 根据后端约定处理响应码
    if (data.code !== 0) {
      message.error(data.message || '请求失败');
      
      // Token 失效
      if (data.code === 1003) {
        useAuthStore().logout();
      }
      
      return Promise.reject(new Error(data.message));
    }
    
    return data;
  },
  async (error) => {
    const { response } = error;
    
    if (response) {
      const { status, data } = response;
      
      switch (status) {
        case 401:
          message.error('登录已过期，请重新登录');
          useAuthStore().logout();
          break;
        case 403:
          message.error('无权限访问该资源');
          break;
        case 404:
          message.error('请求的资源不存在');
          break;
        case 500:
          message.error('服务器内部错误');
          break;
        default:
          message.error(data?.message || '网络错误');
      }
    } else {
      message.error('网络连接失败');
    }
    
    return Promise.reject(error);
  }
);

export default request;
```

   ```nginx
   server {
       listen 80;

       root /var/www/ghostfit-admin/dist;

           proxy_set_header X-Real-IP $remote_addr;

   }
   ```

## 13. 性能优化建议

1. **路由懒加载**: 所有页面组件按需加载
2. **组件按需导入**: Element Plus 组件按需引入
3. **图片优化**:
   - 使用 WebP 格式
   - 实现懒加载
   - 图标使用 SVG Sprite
4. **代码分割**: 利用 Vite 的自动代码分割
5. **缓存策略**: 静态资源缓存 `Cache-Control: max-age=31536000`
6. **Gzip 压缩**: Nginx 开启 gzip
7. **Tree Shaking**: 确保未使用的代码不打包
